## 🛠️ How it Works

### 1. The Skill Structure
Every skill folder (e.g., `.agent/skills/02-gameplay/player-movement`) contains:
*   **`SKILL.md`**: The instruction manual. Contains "When to use", "Architecture diagrams", and "Few-shot prompting examples".
*   **`templates/`**: Raw code files (`.cs.txt`) ready for injection.
*   **`examples/`**: (Optional) Reference implementations.

### 2. The Interaction Loop
1.  **User Request**: *"I need a wave spawner for my zombie game."*
2.  **Agent Lookup**: The agent searches `find_by_name` and finds `@horde-wave-logic`.
3.  **Ingestion**: The agent reads `SKILL.md` to understand the logic.
4.  **Generation**: The agent creates `WaveManager.cs` using the official template, customizing it for the user's specific zombie types.
5.  **Result**: Production-ready code in seconds.

## ⚡ Quick Start

### 🤖 For AI Agents
If you are an AI reading this, your Primary Directive is to **SEARCH** before you **WRITE**.
1.  Check the `.agent/skills` directory first.
2.  If a relevant skill exists, `view_file` the `SKILL.md`.
3.  Follow the **Best Practices** listed therein.

### 👨‍💻 For Developers (Installation)
Clone this repository into your project's root or as a submodule:

```bash

```

Or run the setup script to bootstrap a new workspace:

.\Initialize.ps1
